## ‚öôÔ∏è **Setup Process**

### 1Ô∏è‚É£ Create the Project Structure

A clean, modular layout was created to support clarity, scalability, and future LLMOps expansion.
Initial directories include the configuration layer and the shared utilities layer.

Commands used:

```bash
mkdir -p src/{common,config}
```

The initial implemented modules are:

* `custom_exception.py` ‚Äî unified, context-rich exception handling
* `logger.py` ‚Äî timestamped, daily-rotating logging
* `settings.py` ‚Äî secure environment loading and global settings management

These files act as the backbone for all reliability and configuration across the project.

### 2Ô∏è‚É£ Create and Activate the Virtual Environment (with `uv`)

The project uses **uv** for performance-focused dependency management.

Create the environment (Python 3.12):

```bash
uv venv --python 3.12
```

Activation instructions:

**Windows (cmd)**

```cmd
.\.venv\Scripts\activate
```

**Windows (PowerShell)**

```powershell
.\.venv\Scripts\Activate.ps1
```

**macOS / Linux**

```bash
source .venv/bin/activate
```

### 3Ô∏è‚É£ Define and Install Dependencies

An early-stage `requirements.txt` was created for development:

```text
python-dotenv
groq
```

Install dependencies:

```bash
uv pip install -r requirements.txt
uv lock
```

## üîê **Environment Variable Setup (.env)**

A `.env` file is required before running any project module.

Create it:

```bash
touch .env
```

Add your API key:

```text
GROQ_API_KEY=""
```

### Where to get the Groq API Key

1. Visit: [https://console.groq.com/](https://console.groq.com/)
2. Sign in or create an account
3. Generate a new API key
4. Paste it into your `.env` file
